Структура проекта:

    Серверная часть (Flask):
        Модели данных: одна таблица для книг (SQLite).
        CRUD-операции: добавление, получение списка, обновление, удаление книг.
        Поиск: поддержка поиска по автору, названию, жанру или году.
        Эндпоинт для получения статистики по коллекции (общее количество книг, распределение по жанрам и т.д.).

    Клиентская часть (консольное приложение):
        Меню с вариантами: добавить книгу, обновить книгу, удалить книгу, посмотреть список, поиск, подробная информация, выход.
        Использование REST API для взаимодействия с сервером.
        Обработка ошибок (например, неправильный ввод, ошибки сети).

    Тестовые данные:
        Несколько книг для тестирования (название, автор, жанр, год, описание, количество экземпляров).

    Подготовка к десктопному GUI:
        Отдельная заготовка на основе Tkinter для будущего интерфейса.
